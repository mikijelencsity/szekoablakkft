# Design Principles

## Layout system
- Use strong visual hierarchy, not uniform sections
- Avoid repetitive block structures
- Every section must have a different visual rhythm
- Use contrast in scale, not just color

## Spacing
- Whitespace is a functional element, not decoration
- Large spacing is used to signal importance
- Tight spacing is used for grouped meaning
- Never use uniform vertical spacing across all sections

## Visual hierarchy
Each screen must answer:
1. What is this?
2. Why should I care?
3. What should I do next?

Only one primary focus per viewport.

## Section structure
Preferred pattern:
- Emotional hook
- Context explanation
- Proof (images, results, real work)
- Action (CTA)

## Grid rules
- Use flexible grid, not rigid symmetry
- Break grid intentionally for emphasis
- Align elements by intent, not by template

## Content density
- Low density for hero and trust sections
- Medium density for services and explanations
- High density only for structured data (pricing, specs)

## Visual rhythm
- Alternate between full-width and contained sections
- Alternate between image-led and text-led sections
- Avoid predictable repetition

## Golden rule
If two sections feel similar, one of them is wrong.
