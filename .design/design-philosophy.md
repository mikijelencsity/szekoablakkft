# Design Philosophy

## Core idea
We do not build websites. We design perceived trust, clarity, and conversion.

Every interface exists for one purpose: to turn attention into action without friction.

## Target outcome
A visitor should feel in under 5 seconds:
- this company is real
- this company is experienced
- this company delivers quality work
- contacting them is the obvious next step

## What we optimize for
1. Trust
2. Clarity
3. Conversion
4. Perceived quality

Visual "wow" is only valuable if it reinforces trust or understanding.

## What we avoid
- decorative animations with no meaning
- cluttered layouts
- equal visual weight for all elements
- template-like structures
- unnecessary complexity

## Brand positioning
The interface should feel:
- premium but not luxurious in a detached way
- human, local, and reliable
- modern but not experimental

Think: Apple-level clarity applied to real-world service businesses.

## Design principle hierarchy
If conflicts occur:
1. Conversion wins over aesthetics
2. Clarity wins over creativity
3. Trust wins over novelty
4. Simplicity wins over density

## Core belief
A good website does not impress first.
It convinces first.
Impresses second.
